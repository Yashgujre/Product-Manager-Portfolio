# Product Management Portfolio Build Plan

## Goal
Build a strong portfolio website on GitHub first, then add content (blogs + case studies), then add working product demo apps (web, iOS-style, Android) that recruiters and hiring managers can click and try.

## Constraints
- Use only free tools and free hosting.
- Execute everything from this chat workflow.
- Prioritize practical, recruiter-friendly outcomes over unnecessary complexity.

## Phase 1: Website Foundation (Start Here)
Objective: Launch a clean, professional portfolio website hosted with GitHub.

### Deliverables
- Portfolio website codebase in this repository.
- Initial pages:
  - Home
  - About
  - Experience
  - Projects (placeholder for future apps)
  - Blog
  - Case Studies
  - Contact
- Responsive design (desktop + mobile).
- Deployed via GitHub Pages (or equivalent free GitHub-based hosting).

### Decisions
- Keep stack simple and free.
- Optimize for fast loading and clear PM storytelling.
- Make navigation easy for recruiter, hiring manager, director/VP audiences.

## Phase 2: Add Core Information
Objective: Fill the website with professional PM profile content.

### Deliverables
- Strong positioning statement (Product Manager, 5+ years, US market context).
- Experience summaries focused on outcomes, metrics, and decisions.
- Skills section (discovery, prioritization, analytics, experimentation, execution).
- Resume download link.
- Contact and social links (LinkedIn, GitHub, etc.).

### Content Rules
- Use concise, business-facing language.
- Highlight impact with measurable outcomes when available.
- Keep each section skimmable in under 30 seconds.

## Phase 3: Blogs + Case Studies
Objective: Publish written proof of product thinking and execution maturity.

### Deliverables
- 5-10 blogs over time.
- 3-5 detailed PM case studies.
- Dedicated templates for consistency.

### Blog Topics (Initial)
- Product strategy and prioritization frameworks.
- Metrics design and north-star thinking.
- Experiment design and tradeoff decisions.
- Stakeholder management in cross-functional teams.
- Lessons learned from shipping ambiguous products.

### Case Study Structure
1. Problem context
2. User/customer insight
3. Product strategy and options considered
4. Roadmap and prioritization logic
5. Execution with engineering/design/data
6. Results and metrics
7. What I would improve next

## Phase 4: Working App Demos
Objective: Build interactive demos that visitors can click and try from portfolio links.

### Deliverables
- Web app demos (hosted free).
- Mobile demos with one codebase where possible.
- Links embedded on Projects page.

### Practical Free Approach
- Start with web apps first.
- Build mobile apps with React Native + Expo to target both iOS and Android efficiently.
- Publish Android installable artifact where feasible (free path).
- For iOS, provide browser-playable demo and walkthrough if App Store distribution cost is not desired.

### Demo Requirements
- Clear product problem statement.
- Working core flow (not just static UI).
- Basic analytics/events simulation.
- Short "PM lens" notes: objective, KPIs, tradeoffs.

## Phase 5: Portfolio Positioning for Hiring Audiences
Objective: Make each artifact easy to evaluate for different seniority levels.

### Recruiter View
- Fast overview: role fit, domain exposure, impact highlights.

### Hiring Manager View
- Product depth: decisions, prioritization quality, execution rigor.

### Director/VP View
- Strategic thinking: market framing, leverage, scalable outcomes.

## Phase 6: Ongoing Iteration Loop
Objective: Continuously improve portfolio quality.

### Loop
1. Add one new artifact (blog, case study, or app)
2. Improve one existing artifact
3. Tighten messaging based on target roles
4. Keep all links live and tested

## Execution Order (Confirmed)
1. Create website
2. Put profile information on it
3. Add blogs and case studies
4. Build working apps and integrate links
5. Keep improving quality and positioning

## Definition of Done (Portfolio v1)
- Live GitHub-hosted website.
- Complete PM profile content.
- At least 3 published blogs.
- At least 2 case studies.
- At least 1 working app demo link.
- Clear contact path for recruiters/hiring managers.
